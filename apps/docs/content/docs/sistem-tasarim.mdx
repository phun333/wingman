Sistem tasarımı mülakatları, büyük ölçekli yazılım sistemlerinin nasıl tasarlanacağını test eder. Wingman, bu tür mülakatlar için **interaktif whiteboard** desteği sunar.

## Mülakat Akışı

<Steps>
  <Step number={1} title="Gereksinim Analizi (5 dk)">
    AI mülakatçı bir problem tanımlar. Fonksiyonel ve non-fonksiyonel gereksinimleri netleştir.

    ```
    Örnek sorular:
    → "Twitter benzeri bir sosyal medya platformu tasarla"
    → "URL kısaltma servisi tasarla"
    → "Gerçek zamanlı chat uygulaması tasarla"

    Sorulacak sorular:
    → "Günlük kaç kullanıcı bekliyoruz?"
    → "Okuma/yazma oranı nedir?"
    → "Gecikme toleransı ne kadar?"
    ```
  </Step>
  <Step number={2} title="Üst Düzey Tasarım (10 dk)">
    Whiteboard'da sistemin genel mimarisini çiz:

    ```
    Client → Load Balancer → API Gateway
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼             ▼
              Auth Service  Core Service  Notification
                    │             │             │
                    ▼             ▼             ▼
                User DB      Main DB       Queue (Kafka)
                              Cache (Redis)
    ```
  </Step>
  <Step number={3} title="Detaylı Tasarım (15 dk)">
    Kritik bileşenleri derinleştir — veritabanı şeması, API endpoint'leri, ölçeklendirme stratejisi.
  </Step>
  <Step number={4} title="Trade-off Tartışması (5 dk)">
    Tasarım kararlarının artılarını ve eksilerini tartış.
  </Step>
</Steps>

## Trade-off Örnekleri

| Karar | Seçenek A | Seçenek B | Durum |
|-------|-----------|-----------|-------|
| **Veritabanı** | SQL (PostgreSQL) | NoSQL (MongoDB) | İlişkisel veri → SQL, esnek şema → NoSQL |
| **Mimari** | Monolith | Microservice | Küçük ekip → Monolith, ölçek → Microservice |
| **Mesajlaşma** | Push (WebSocket) | Pull (Polling) | Gerçek zamanlı → Push, basitlik → Pull |
| **Cache** | Write-through | Write-back | Tutarlılık → Through, performans → Back |

<Callout type="tip" title="Basit Başla">
  İlk önce basit bir tasarımla başla, sonra ölçeklendir. "Premature optimization is the root of all evil."
</Callout>

<Callout type="tip" title="Rakamları Kullan">

</Callout>
